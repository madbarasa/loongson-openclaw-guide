龙芯3A5000 芯片实拍与规格

系统安装（Deepin LoongArch）

推荐系统： Deepin 或 UOS 的 LoongArch 版本，这两款国产系统对龙芯支持最好。
图3：Deepin LoongArch 安装界面（与x86版本类似）
1

下载镜像

访问 Deepin 官网下载 LoongArch 架构镜像：

wget https://cdimage.deepin.com/loongarch/

文件名通常包含 "deepin" 和 "loongarch" 关键字

2

制作启动U盘（在x86电脑上操作）

# 查看U盘设备名
lsblk

# 写入镜像（/dev/sdX替换为你的U盘设备）
sudo dd if=deepin-loongarch.iso of=/dev/sdX bs=4M status=progress
sudo sync
3

启动并安装系统

  • U盘插入龙芯 mini PC
  • 开机按 F12（或Del）进入启动菜单
  • 选择U盘启动项
  • 按照图形界面提示完成分区、安装
4

验证架构

# 重启后登录终端
uname -m

正确输出： loongarch64

Node.js 20.x 安装（两种方法）

注意： LoongArch 架构需要特殊的 Node.js 二进制包。 官方尚未提供预编译版本，需要从源码编译或使用社区构建的包。

方法A：手动解压预编译包（如有）

cd /tmp
wget https://nodejs.org/dist/v20.11.1/node-v20.11.1-linux-loongarch64.tar.xz
tar -xf node-v20.11.1-linux-loongarch64.tar.xz
sudo mv node-v20.11.1-linux-loongarch64 /opt/nodejs

# 创建软链接
sudo ln -sf /opt/nodejs/bin/node /usr/local/bin/node
sudo ln -sf /opt/nodejs/bin/npm /usr/local/bin/npm
sudo ln -sf /opt/nodejs/bin/npx /usr/local/bin/npx

如果官网没有 loongarch64 版本，跳过此方法，使用方法B

方法B：从源码编译（保证成功）

git clone https://github.com/nodejs/node.git
cd node
git checkout v20.11.1

./configure --without-intl
make -j$(nproc)

sudo make install

⏱️ 预计耗时 1-2 小时（取决于CPU性能）

# 验证安装成功
node --version  # 应显示 v20.x.x
npm --version   # 应显示 10.x.x

故障排除 & 调优

问题现象 可能原因与解决方案
npm ERR! 安装失败 原因：LoongArch 包缺失或网络问题
解决：更换 npm 源 npm config set registry https://registry.npmmirror.com
openclaw: command not found 原因：npm 全局路径未加入 PATH
解决： 
sudo npm config set prefix /usr/local
然后重新安装
服务启动后立即退出 原因：端口被占用或权限不足
解决： 
sudo lsof -i:8080  # 查看占用
sudo kill -9 PID   # 结束进程
网页无法访问 OpenClaw API 原因：CORS 跨域限制
解决：配置 OpenClaw 允许跨域或使用 Nginx 反向代理
Node.js 编译时间过长 原因：龙芯3A5000性能有限
解决：使用 make -j4 启用4核并行编译；或使用预编译包（如有）

💡 极客调优建议

  • 使用 tmux 或 screen 保持会话，避免 SSH 断开导致服务停止
  • 配置 systemd 开机自启 OpenClaw 服务
  • 使用 nginx 反向代理 + SSL 证书，实现 HTTPS 安全访问
  • 定期查看 ~/.openclaw/logs/ 目录下的日志文件
