📖 项目简介

本项目将介绍如何使用基于龙芯3A5000处理器的mini PC，搭建完整的OpenClaw智能助手环境，并将Web界面部署到GitHub Pages，实现远程访问和控制。

💻 硬件准备

龙芯3A5000 mini PC 规格

处理器 龙芯3A5000 (4核, 主频2.3-2.5GHz)
架构 LoongArch (64位)
内存 DDR4 3200 (建议≥8GB)
TDP 35W
封装 BGA1211 (带锡球)
存储 eMMC/SATA/NVMe (建议≥128GB)
💡 提示 龙芯3A5000是真正的国产自主CPU，支持完整的Linux发行版，可以运行Node.js等现代开发环境。

🖥️ 系统安装

1. 安装 LoongArch 架构的 Linux 系统

推荐使用 Deepin 龙芯版 或 UOS 龙芯版，这些系统已针对LoongArch优化：

# 下载 Deepin LoongArch 镜像
wget https://cdimage.deepin.com/loongarch/

# 制作启动U盘（在x86电脑上）
sudo dd if=deepin-loongarch.iso of=/dev/sdX bs=4M status=progress

# 插入U盘，启动mini PC，按提示安装系统

2. 基础环境配置

# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y git curl wget build-essential python3 python3-pip

# 验证架构
uname -m
# 应显示：loongarch64

📦 安装 Node.js

OpenClaw 需要 Node.js 18+ 环境。LoongArch 架构需要单独下载适配版本：

# 方法1：使用Nodesource（如有LoongArch仓库）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 方法2：手动下载预编译二进制
cd /tmp
wget https://nodejs.org/dist/v20.11.1/node-v20.11.1-linux-loongarch64.tar.xz
tar -xf node-v20.11.1-linux-loongarch64.tar.xz
sudo mv node-v20.11.1-linux-loongarch64 /opt/nodejs
sudo ln -s /opt/nodejs/bin/node /usr/local/bin/node
sudo ln -s /opt/nodejs/bin/npm /usr/local/bin/npm

# 验证安装
node --version  # 应显示 v20.x.x
npm --version
⚠️ 注意 如果官方没有LoongArch预编译版本，可能需要从源码编译Node.js，这会消耗较长时间（1-2小时）。

🤖 安装 OpenClaw

# 全局安装 OpenClaw
sudo npm install -g openclaw

# 验证安装
openclaw --version

# 初始化 OpenClaw 配置
openclaw init

# 启动 OpenClaw 服务
openclaw gateway start
✅ 成功 如果看到 "Gateway started"，说明 OpenClaw 已正常运行。

配置 OpenClaw

编辑配置文件 ~/.openclaw/config.json，根据需要调整：

{
  "gateway": {
    "port": 8080,
    "token": "your-secret-token",
    "logLevel": "info"
  },
  "plugins": {
    "lark": true,
    "wecom": false
  }
}

🗂️ 创建 OpenClaw 项目

# 创建项目目录
mkdir ~/my-openclaw && cd ~/my-openclaw

# 初始化项目结构
openclaw project init

# 查看项目文件
tree ~/.openclaw
# 应该看到：
# ~/.openclaw/
# ├── AGENTS.md
# ├── SOUL.md
# ├── USER.md
# ├── MEMORY.md
# ├── HEARTBEAT.md
# ├── TOOLS.md
# ├── skills/
# └── workspace/
📝 提示 首次运行会提示你设置身份信息，按照向导完成即可。

🧪 测试连接

打开浏览器，访问：

http://你的龙芯miniPC的IP:8080

如果看到 OpenClaw 的欢迎页面，说明服务正常运行。

配置反向代理（可选，方便访问）

# 使用 Nginx 反向代理
sudo apt install -y nginx

sudo tee /etc/nginx/sites-available/openclaw << 'EOF'
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/openclaw /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

🚀 部署到 GitHub Pages

将你的 OpenClaw 项目网页化，部署到 GitHub Pages，实现公网访问。

1. 准备 GitHub 仓库

  1. 登录 GitHub，创建新仓库 用户名.github.io（替换<用户名>为你的 GitHub 用户名）
  2. 或创建普通仓库 openclaw-dashboard
  3. 克隆仓库到龙芯mini PC： 
    git clone https://github.com/你的用户名/仓库名.git
cd 仓库名

2. 创建静态网页

在 OpenClaw 项目下创建网页文件（例如 dashboard.html），内容可以是：

# 简单示例：显示OpenClaw状态
cat > ~/my-openclaw/dashboard.html << 'EOF'
<!DOCTYPE html>
<html>
<head>
    <title>OpenClaw Status</title>
    <meta charset="UTF-8">
    <style>
        body { font-family: sans-serif; padding: 20px; }
        .status { color: green; font-weight: bold; }
    </style>
</head>
<body>
    <h1>OpenClaw 运行状态</h1>
    <p>服务状态：<span class="status">✅ 运行中</span></p>
    <p>访问时间：<span id="time"></span></p>
    <script>
        document.getElementById('time').textContent = new Date().toLocaleString();
    </script>
</body>
</html>
EOF

3. 配置 GitHub Pages

# 将网页文件复制到仓库
cp ~/my-openclaw/dashboard.html ~/仓库名/
cd ~/仓库名

# 初始化git（如果还没有）
git init
git add .
git commit -m "Initial: OpenClaw Dashboard"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main

# 在GitHub网页设置Pages
# Settings → Pages → Source: main branch /root folder
# 保存后，访问：https://你的用户名.github.io/仓库名/
🎉 完成！ 你的 OpenClaw 状态页面现在可以通过 GitHub Pages 公网访问了。

🔄 进阶：自动更新网页

可以通过 GitHub Actions 自动更新网页内容，例如每5分钟抓取 OpenClaw 日志并更新：

# 在仓库中创建 .github/workflows/update.yml
cat > .github/workflows/update.yml << 'EOF'
name: Update Dashboard

on:
  schedule:
    - cron: '*/5 * * * *'  # 每5分钟
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Fetch OpenClaw status
        run: |
          # 这里可以调用你的OpenClaw API获取状态
          echo "Last updated: $(date)" > status.txt
      - name: Commit changes
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add .
          git commit -m "Auto update: $(date)" || echo "No changes"
          git push
EOF

这样你的 GitHub Pages 网页就会自动保持最新状态。

问题 解决方案
Node.js 无法安装 从源码编译或寻找 LoongArch 社区维护的包

网页无法访问 检查防火墙是否开放 80/443 端口
