系统安装

推荐使用 Deepin LoongArch 版 或 UOS LoongArch 版，对龙芯优化最好。

步骤1：制作启动盘

1

下载LoongArch镜像

wget https://cdimage.deepin.com/loongarch/

文件名通常包含 "deepin" 和 "loongarch" 关键字

2

写入U盘（在x86电脑上操作）

# 查看U盘设备名
lsblk

# 写入镜像（/dev/sdX替换为你的U盘）
sudo dd if=deepin-loongarch.iso of=/dev/sdX bs=4M status=progress
sudo sync
3

启动并安装

  • U盘插入龙芯mini PC
  • 开机按F12（或其他键）进入启动菜单
  • 选择U盘启动
  • 按照图形界面提示完成安装
验证 安装完成后，打开终端执行 uname -m，应显示 loongarch64。

Node.js 安装

龙芯架构需要特殊处理的 Node.js：

方法A：使用预编译包（推荐）

cd /tmp
wget https://nodejs.org/dist/v20.11.1/node-v20.11.1-linux-loongarch64.tar.xz
tar -xf node-v20.11.1-linux-loongarch64.tar.xz
sudo mv node-v20.11.1-linux-loongarch64 /opt/nodejs

# 创建软链接
sudo ln -sf /opt/nodejs/bin/node /usr/local/bin/node
sudo ln -sf /opt/nodejs/bin/npm /usr/local/bin/npm
sudo ln -sf /opt/nodejs/bin/npx /usr/local/bin/npx

方法B：从源码编译

# 如果预编译包不可用
git clone https://github.com/nodejs/node.git
cd node
./configure
make -j$(nproc)
sudo make install

⏳ 预计耗时 1-2 小时

# 验证安装
node --version  # v20.x.x
npm --version   # 10.x.x

OpenClaw 部署

确保已设置 npm 全局安装路径权限，或使用 sudo npm install -g

1. 全局安装

sudo npm install -g openclaw

2. 初始化配置

# 首次运行会生成配置文件
openclaw init
$ openclaw init
[INFO] 初始化 OpenClaw 配置...
[INFO] 创建目录: ~/.openclaw/
[INFO] 生成默认配置文件...
[SUCCESS] 初始化完成！
$

3. 启动服务

# 启动网关服务（默认端口8080）
openclaw gateway start

# 查看状态
openclaw gateway status

常用命令

  • openclaw gateway stop - 停止服务
  • openclaw gateway restart - 重启服务
  • openclaw logs - 查看日志
  • openclaw plugins list - 列出插件

部署到 GitHub Pages

目标：将 OpenClaw 控制界面发布到 GitHub Pages，实现公网访问。

1. 准备网页文件

创建一个简单的状态展示页面 status.html：

mkdir ~/openclaw-web && cd ~/openclaw-web
cat > index.html << 'EOF'
<!DOCTYPE html>
<html>
<head>
    <title>OpenClaw Status</title>
    <meta charset="UTF-8">
    <style>
        body { 
            font-family: sans-serif; 
            background: #0a0a0f; 
            color: #00ff88; 
            padding: 40px;
        }
        .status { 
            font-size: 2rem; 
            margin: 20px 0;
        }
        .online { color: #00ff88; }
        .offline { color: #ff4444; }
    </style>
</head>
<body>
    <h1>🤖 OpenClaw 服务状态</h1>
    <p class="status online">✅ 运行正常</p>
    <p>最后更新：<span id="time"></span></p>
    <script>
        document.getElementById('time').textContent = new Date().toLocaleString();
        // 这里可以调用 OpenClaw API 获取真实状态
    </script>
</body>
</html>
EOF

2. 创建 GitHub 仓库

1

创建仓库

登录 GitHub → New repository → 仓库名：用户名.github.io（或任意名称）

2

推送代码

cd ~/openclaw-web
git init
git add .
git commit -m "Initial: OpenClaw Dashboard"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main

3. 启用 Pages

  1. 进入仓库 Settings → Pages
  2. Source 选择 "main branch" 和 "/root folder"
  3. 点击 Save
  4. 等待 1-2 分钟
访问地址：https://你的用户名.github.io/仓库名/

进阶：API 实时数据

让网页实时显示 OpenClaw 状态（消息数、插件状态等）：

OpenClaw API 示例

// status.html 中的 JavaScript
async function fetchOpenClawStatus() {
    try {
        const response = await fetch('http://localhost:8080/api/status', {
            headers: {
                'Authorization': 'Bearer YOUR_TOKEN'
            }
        });
        const data = await response.json();
        
        document.getElementById('status-text').textContent = 
            data.online ? '🟢 在线' : '🔴 离线';
        document.getElementById('plugins-count').textContent = data.plugins.length;
        document.getElementById('messages-count').textContent = data.messagesTotal;
    } catch (error) {
        console.error('Failed to fetch:', error);
    }
}

// 每10秒刷新
setInterval(fetchOpenClawStatus, 10000);
fetchOpenClawStatus();
安全提醒：API 需要认证，不要在公开页面暴露敏感 token。建议使用后端代理或只显示非敏感信息。

故障排除

问题现象 可能原因 解决方案
Node.js 无法安装 LoongArch 包缺失 从源码编译或使用社区维护的包
openclaw 命令找不到 npm 全局路径未配置 sudo npm config set prefix /usr/local
服务启动失败 端口被占用 sudo lsof -i:8080 查看并杀进程
GitHub Pages 404 未正确配置 Pages 检查 Settings → Pages 的 branch/path
网页无法访问 OpenClaw API CORS 跨域问题 配置 OpenClaw 允许跨域或使用代理
