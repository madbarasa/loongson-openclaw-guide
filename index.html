龙芯3A5000：国产芯的骄傲

Deepin LoongArch 系统安装

推荐系统： Deepin LoongArch 或 UOS LoongArch 对龙芯支持最好，开箱即用。
FIG 3. Deepin LoongArch 安装界面（分区设置阶段）
01

下载 LoongArch 镜像

访问 Deepin 官网或镜像站，获取 LoongArch 架构的 ISO 镜像：

FIG 3.0. Deepin LoongArch ISO 下载完成（文件大小约 4GB） 
# 示例命令（实际URL请访问官网获取）
wget https://cdimage.deepin.com/loongarch/
02

制作启动U盘

在 x86 电脑上使用 dd 命令写入U盘：

FIG 3.1. 使用 lsblk 确认 U 盘设备（如 /dev/sdb） 
# 查看U盘设备
lsblk

# 写入（/dev/sdX 替换为你的U盘设备）
sudo dd if=deepin-loongarch.iso of=/dev/sdX bs=4M status=progress
sudo sync
FIG 3.2. dd 写入过程（bs=4M 加速，status=progress 显示进度）
03

启动并安装

FIG 3.3. 启动菜单（按 F12 选择 U 盘启动）
  • U盘插入龙芯 mini PC
  • 开机按 F12 或 Del 进入启动菜单
  • 选择 U盘 启动项
  • 按图形界面提示完成分区与安装
04

分区设置

FIG 3.4. 分区设置界面（建议 20GB 以上 root 分区）

在安装向导中选择"专家分区"或"自动分区"，建议配置：

  • / (root) - 20GB+，ext4 文件系统
  • swap - 内存的 1-2 倍（8GB 内存可配 8GB swap）
  • /home - 剩余空间，ext4
05

等待安装

FIG 3.5. 系统安装进度（预计 10-20 分钟）

安装过程中会下载必要组件，保持网络畅通。安装完成后重启。

重要： LoongArch 架构需要特别处理。官方暂未提供预编译包， 需要从源码编译或寻找社区构建版本。推荐方法B（源码编译），最稳定。

方法A：预编译包（如有）

FIG 4.2. 下载 Node.js LoongArch 预编译包 
cd /tmp
wget https://nodejs.org/dist/v20.11.1/node-v20.11.1-linux-loongarch64.tar.xz
tar -xf node-v20.11.1-linux-loongarch64.tar.xz
sudo mv node-v20.11.1-linux-loongarch64 /opt/nodejs

# 创建软链接
sudo ln -sf /opt/nodejs/bin/node /usr/local/bin/node
sudo ln -sf /opt/nodejs/bin/npm /usr/local/bin/npm
sudo ln -sf /opt/nodejs/bin/npx /usr/local/bin/npx

故障排除 & 极客调优

FIG 6.0. 常见错误提示（npm 安装失败、端口占用等）
FIG 6.1. npm ERR! 详细错误信息（LoongArch 包缺失）
FIG 6.2. 端口 8080 被占用（使用 lsof -i:8080 排查）
问题现象 解决方案
npm ERR! 安装失败 原因：LoongArch 包缺失或网络超时
解决：更换 npm 源
npm config set registry https://registry.npmmirror.com
openclaw: command not found 原因：npm 全局路径未加入 PATH
解决： 
sudo npm config set prefix /usr/local
然后重新安装
服务启动后立即退出 原因：端口占用或权限不足
解决： 
sudo lsof -i:8080  # 查看占用进程
sudo kill -9 PID   # 结束进程
Node.js 编译耗时过长 原因：龙芯性能有限，单核编译慢
解决：使用 make -j4 启用4核并行编译
网页无法访问 API 原因：CORS 跨域限制
解决：配置 OpenClaw 允许跨域或使用 Nginx 反向代理

$ 极客调优建议

  • 使用 tmux 或 screen 保持会话，SSH 断开后服务不中断
  • 编写 systemd service 文件，实现 OpenClaw 开机自启
  • 配置 Nginx 反向代理 + SSL，实现 HTTPS 安全访问
  • 定期查看 ~/.openclaw/logs/ 目录下的日志文件
  • 使用 htop 监控系统资源占用情况
